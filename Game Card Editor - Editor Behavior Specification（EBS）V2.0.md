# Game Card Editor
# Editor Behavior Specification（EBS）

**Version：** V2.0  
**Document Type：** Editor Behavior Specification（EBS）

---

# 1. 文档说明

本文档定义全球杀卡牌编辑器（Global Kill Card Editor）的编辑行为规范（Editor Behavior Specification，EBS），用于统一编辑器中所有对象的操作逻辑、交互规则及状态变化。

SDS 负责定义系统结构与功能边界，EBS 则负责定义编辑器内部每一个对象如何响应用户操作、如何改变状态以及如何反馈结果。开发过程中，所有编辑行为均应遵循本规范，保证整个编辑器具有一致、稳定、易理解的交互体验。

---

# 2. 编辑器设计原则

本编辑器定位为面向普通用户及青少年使用的轻量化卡牌编辑器，而不是专业设计软件。因此，系统应始终坚持"简单、直观、低学习成本"的设计原则。

整个编辑器仅围绕三个核心概念展开：

- 固定尺寸画布（Canvas）
- 图片对象（Image Object）
- 文字对象（Text Object）

所有对象均可通过统一方式进行选择、编辑、排序、保存和导出，不因对象类型而改变基本操作逻辑。

编辑器遵循以下原则：

- 所见即所得（WYSIWYG）。
- 所有修改实时显示。
- 所有操作均可撤销与重做。
- 相同对象采用相同操作方式。
- 用户始终关注内容，而不是软件本身。

---

# 3. 编辑器工作流程

整个编辑过程遵循固定流程：

**创建项目 → 创建卡牌（或选择模板） → 添加图片对象 → 添加文字对象 → 调整对象位置与层级 → 保存卡牌 → 导出 PNG → 自动生成缩略图 → 后续继续编辑**

所有流程均围绕单张卡牌展开，不涉及游戏逻辑。

---

# 4. 编辑器状态

编辑器始终处于以下四种状态之一。

| 状态 | 说明 |
|------|------|
| Idle | 当前没有任何对象被选中 |
| Selected | 一个对象被选中 |
| Editing | 正在编辑文字内容 |
| Transforming | 正在移动、缩放或旋转对象 |

任何时刻仅允许一种状态处于激活状态。

---

# 5. 画布行为

每张卡牌对应一个固定尺寸画布。

画布尺寸在创建卡牌时确定，后续仅允许缩放查看，不允许修改实际尺寸。

画布支持以下行为：

- 鼠标滚轮缩放查看。
- 按住空格键拖动画布。
- 双击画布空白区域恢复适合窗口大小。
- 点击空白区域取消当前对象选择。

对象允许超出画布范围进行编辑，但导出图片时仅输出画布范围内内容。

---

# 6. 对象管理

编辑器不使用复杂的"图层"概念，而统一采用"对象（Object）"作为编辑单位。

对象分为两种类型：

| 类型 | 说明 |
|------|------|
| 图片对象（Image Object） | 背景、边框、角色、图标、装饰等所有图片元素 |
| 文字对象（Text Object） | 名称、说明、数值、标签等所有文字内容 |

编辑器不会理解对象的业务含义，仅负责对象的显示、编辑与管理。

---

# 7. 对象列表（Object List）

对象列表位于编辑器右侧，是整个编辑器的核心管理区域。

对象列表用于管理当前卡牌中的全部对象，并决定对象在画布中的前后显示顺序。列表最上方表示最上层对象，列表最下方表示最底层对象。

对象列表采用类似 Photoshop、Illustrator、Figma 的图层管理方式，但交互大幅简化，仅保留最常用功能。

对象列表支持以下能力：

| 功能 | 说明 |
|------|------|
| 选择对象 | 点击对象立即选中，同时画布同步高亮 |
| 调整顺序 | 拖拽对象上下移动，实时改变显示层级 |
| 显示 / 隐藏 | 点击眼睛图标控制显示状态 |
| 锁定 / 解锁 | 点击锁图标控制是否允许编辑 |
| 重命名 | 双击对象名称修改名称 |
| 删除 | 删除当前对象 |
| 复制 | 复制当前对象 |
| 新建 | 新建图片对象或文字对象 |

对象列表中的每个对象建议显示以下信息：

- 对象缩略图（图片对象显示图片预览，文字对象显示"T"图标）
- 对象名称
- 显示 / 隐藏按钮
- 锁定按钮

例如：

```text
🖼 卡牌背景
🖼 卡牌边框
🖼 角色图片
📝 卡牌名称
📝 技能说明
🖼 阵营图标
```

对象名称可自动生成，也允许用户修改。

画布与对象列表始终保持同步。

点击对象列表中的对象，画布立即选中对应对象；点击画布中的对象，对象列表自动定位到对应对象。

拖拽调整对象顺序时，画布实时刷新显示结果。

整个对象列表不提供混合模式、通道、蒙版等专业设计功能，以保证操作简单直观。

---

# 8. 图片对象行为

图片对象负责显示所有图片资源。

图片上传后自动保存至当前 Project 的 Assets 中，并建立资源索引。

图片对象支持以下行为：

- 添加图片
- 替换图片
- 删除对象
- 拖动位置
- 修改尺寸
- 等比例缩放
- 自由旋转
- 修改透明度
- 锁定
- 隐藏
- 调整对象顺序
- 复制对象

替换图片时，仅替换图片资源，不改变对象其它属性。

删除图片对象时，仅删除当前对象，不删除素材库中的原始图片。

---

# 9. 文字对象行为

文字对象负责显示所有文本内容。

双击文字对象进入编辑状态，编辑完成后点击空白区域或按 Enter 键结束编辑。

文字对象支持以下属性：

- 文本内容
- 字体
- 字号
- 字重
- 字体颜色
- 对齐方式
- 行高
- 字间距
- 文本框尺寸
- 旋转角度
- 透明度

文字修改完成后立即刷新画布。

---

# 10. 对象选择行为

点击任意对象后立即进入选中状态。

对象被选中后，应显示：

- 蓝色选中边框
- 八个缩放控制点
- 一个旋转控制柄（如启用旋转）
- 当前对象尺寸信息

再次点击已选中对象，不改变状态。

点击画布空白区域取消所有对象选择。

V2.0 默认仅支持单对象选择，多对象选择放入后续版本。

---

# 11. 对象移动、缩放与旋转

对象移动时，仅修改坐标位置，不改变其它属性。

对象缩放时，仅改变显示尺寸，不修改图片原始分辨率。

对象旋转时，以对象中心点作为默认旋转中心。

所有变换均应实时刷新画布，并在操作结束后生成一条 Undo 历史记录，而不是连续记录每一次鼠标移动。

---

# 12. 属性面板

属性面板位于编辑器右侧，对象列表下方，根据当前选中对象自动切换内容。

| 当前对象 | 面板内容 |
|----------|----------|
| 无对象 | 画布属性 |
| 图片对象 | 图片属性、尺寸、位置、透明度等 |
| 文字对象 | 文本内容、字体、字号、颜色、对齐方式等 |

属性面板与画布保持实时同步。任何一方发生修改，另一方立即更新显示。

---

# 13. Undo / Redo 行为

所有编辑操作均进入历史记录，包括：

- 创建对象
- 删除对象
- 拖动
- 缩放
- 旋转
- 修改图片
- 修改文字
- 修改属性
- 修改对象顺序

连续拖动、连续缩放和连续旋转均视为一次操作。

建议至少支持 100 步历史记录。

---

# 14. 保存行为

系统支持自动保存和手动保存。

建议在以下情况下自动保存：

- 停止编辑数秒后
- 导出前
- 切换卡牌前
- 关闭页面前

保存内容包括：

- 所有对象
- 对象顺序
- 图片引用
- 文字内容
- 编辑状态
- 当前缩放比例（可选）

重新打开卡牌时，应恢复到保存时状态。

---

# 15. 导出行为

导出时仅输出当前画布范围内所有可见对象。

隐藏对象不参与导出，锁定对象正常参与导出。

导出完成后自动生成以下文件：

| 文件 | 用途 |
|------|------|
| export.png | 高清正式图片 |
| preview.png | 卡牌缩略图 |

缩略图自动保存至当前 Project，用于项目首页浏览和继续编辑。

---

# 16. 删除行为

删除对象无需二次确认。

删除后立即进入 Undo 历史，可通过 Undo 恢复。

删除卡牌时，应同时删除该卡牌的编辑数据、缩略图及导出图片。

素材库中的图片仅在没有任何卡牌引用时，才允许彻底删除。

---

# 17. 异常处理

编辑器应保证任何异常都不会导致编辑数据丢失。

发生异常时，应自动保存当前状态，并提示用户错误信息。重新进入项目后，应优先恢复最近一次自动保存内容。

任何单个对象的数据异常都不得导致整个项目无法打开。

---

# 18. 编辑器行为设计原则

整个编辑器应始终围绕"固定尺寸画布 + 图片对象 + 文字对象"这一核心模型展开。

所有对象均采用统一的编辑逻辑，不因对象的业务用途而改变交互方式。编辑器负责的是对象的排版、组合和导出，而不是理解卡牌规则。

所有设计均应坚持以下原则：

- 操作简单，尽量减少学习成本。
- 行为统一，避免不同对象采用不同交互方式。
- 修改即时反馈，保证编辑体验流畅。
- 数据安全，任何操作均可恢复。
- 界面清晰，让用户始终关注卡牌内容而不是软件复杂性。

本规范作为编辑器行为的唯一标准，后续新增功能均应遵循本规范的设计思想，在保持整体一致性的前提下逐步扩展功能，而不改变编辑器的核心交互模型。
