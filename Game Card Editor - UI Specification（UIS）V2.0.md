# Game Card Editor
# UI Specification（UIS）

**Version：** V1.0  
**Document Type：** User Interface Specification（UIS）

---

# 1. 文档说明

本文档定义 Game Card Editor（游戏卡牌编辑器）的用户界面规范（User Interface Specification，UIS），包括页面布局、界面组成、视觉层级、控件行为及设计原则。

UIS 仅关注"界面应该长什么样"，不描述具体编辑行为。对象如何编辑、如何拖拽、如何保存等内容由《Editor Behavior Specification（EBS）》定义。

整个界面设计应遵循"简单、直观、低学习成本"的原则，让没有任何设计软件经验的用户也能够快速上手。

---

# 2. UI 设计目标

Game Card Editor 并不是 Photoshop、Illustrator 或 Figma，而是一款面向卡牌创作的轻量级编辑器。

因此，整个界面应遵循以下设计原则：

- 保持页面简洁，不出现专业设计软件的大量工具。
- 常用功能始终可见，减少菜单层级。
- 编辑区域保持最大可视面积。
- 操作路径尽量控制在两步以内。
- 所有按钮采用统一视觉语言。
- 重要操作优先使用图标配合文字说明。
- 支持鼠标操作，不依赖快捷键完成核心功能。

---

# 3. 整体布局

编辑器采用经典三栏布局，中间画布始终保持视觉中心。

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部工具栏                                                   │
├────────────┬──────────────────────────────┬──────────────────┤
│            │                              │                  │
│            │                              │                  │
│ 左侧资源区 │          画布编辑区           │ 右侧属性管理区   │
│            │                              │                  │
│            │                              │                  │
├────────────┴──────────────────────────────┴──────────────────┤
│ 底部状态栏                                                   │
└──────────────────────────────────────────────────────────────┘
```

各区域职责明确，不建议用户自行改变布局。

---

# 4. 页面组成

编辑器由五个主要区域组成。

| 区域 | 功能 |
|------|------|
| 顶部工具栏 | 全局操作入口 |
| 左侧资源区 | 模板、素材及项目资源 |
| 中央画布 | 卡牌编辑区域 |
| 右侧管理区 | 对象列表与属性面板 |
| 底部状态栏 | 当前状态及缩放信息 |

---

# 5. 顶部工具栏

顶部工具栏提供整个项目的全局操作，应始终固定显示。

建议包含以下功能：

| 分类 | 功能 |
|------|------|
| 项目 | 新建、打开、保存 |
| 编辑 | Undo、Redo |
| 对象 | 添加图片、添加文字 |
| 导出 | 导出 PNG |
| 视图 | 缩放、适应窗口 |
| 帮助 | 设置、关于 |

按钮采用图标加文字形式，例如：

```text
📂 打开    💾 保存    ↶ 撤销    ↷ 重做

🖼 图片    📝 文字

📤 导出 PNG
```

避免采用多级菜单，常用功能应始终可见。

---

# 6. 左侧资源区

左侧资源区负责管理项目资源。

采用分页（Tab）方式切换。

建议包含以下四个标签：

| 标签 | 内容 |
|------|------|
| Project | 当前项目信息 |
| Templates | 模板库 |
| Assets | 图片素材库 |
| Cards | 当前项目所有卡牌 |

---

## 6.1 Templates

模板以卡片形式展示。

每个模板显示：

- 缩略图
- 模板名称
- 分类

支持：

- 搜索
- 分类筛选
- 双击创建卡牌

---

## 6.2 Assets

素材采用缩略图网格显示。

支持：

- 上传图片
- 删除图片
- 重命名
- 搜索
- 分类

图片支持直接拖拽至画布创建图片对象。

---

## 6.3 Cards

当前项目所有卡牌均采用缩略图方式显示。

每张卡片显示：

- 缩略图
- 卡牌名称
- 最后修改时间

支持：

- 双击打开
- 重命名
- 删除
- 复制

---

# 7. 中央画布

中央区域是整个编辑器最重要的区域。

画布始终保持视觉中心。

建议采用浅灰色工作区搭配白色画布。

```text
────────────────────────────

      灰色工作区

     ┌─────────────┐
     │             │
     │   卡牌画布   │
     │             │
     └─────────────┘

────────────────────────────
```

画布支持：

- 鼠标滚轮缩放
- 中键拖动画布
- 双击恢复适合窗口

画布外区域仅作为工作区，不参与导出。

---

# 8. 右侧管理区

右侧管理区采用上下布局，由"对象列表"和"属性面板"组成。

```text
┌────────────────────────────┐
│       对象列表             │
├────────────────────────────┤
│                            │
│      属性面板              │
│                            │
└────────────────────────────┘
```

对象列表建议占约 40% 高度，属性面板占约 60%。

支持上下拖动调整比例。

---

# 9. 对象列表

对象列表是整个编辑器最重要的管理入口。

对象列表按照画布渲染顺序排列。

列表顶部对象显示在画布最上层。

列表底部对象显示在画布最下层。

每个对象显示以下内容：

- 对象图标
- 缩略图（图片对象）
- 名称
- 显示按钮
- 锁定按钮

示例：

```text
👁 🔒 🖼 卡牌边框

👁 🔓 🖼 角色图片

👁 🔓 📝 卡牌名称

👁 🔓 📝 技能说明

👁 🔓 🖼 阵营图标
```

对象支持直接拖拽排序。

拖动过程中画布实时更新对象层级。

对象名称支持双击修改。

整个对象列表不提供专业设计软件中的混合模式、通道、蒙版等复杂功能。

---

# 10. 属性面板

属性面板根据当前选中对象自动切换内容。

## 未选择对象

显示：

- 画布尺寸
- 缩放比例
- 背景颜色（预留）

## 图片对象

显示：

- 图片预览
- 替换图片
- X、Y 坐标
- 宽度
- 高度
- 旋转角度
- 透明度

## 文字对象

显示：

- 文本内容
- 字体
- 字号
- 字重
- 字体颜色
- 对齐方式
- 行高
- 字间距

所有属性修改后立即刷新画布。

---

# 11. 底部状态栏

底部状态栏用于显示当前工作状态。

建议显示以下信息：

| 内容 | 示例 |
|------|------|
| 当前项目 | 三国杀扩展 |
| 当前卡牌 | 关羽 |
| 缩放比例 | 100% |
| 对象数量 | 12 Objects |
| 保存状态 | 已保存 |

状态栏不提供编辑功能，仅用于状态反馈。

---

# 12. 对话框设计

所有弹窗采用统一风格。

建议用于：

- 新建项目
- 新建卡牌
- 删除确认
- 导出设置
- 设置页面

弹窗尺寸保持适中，不遮挡整个编辑区域。

所有弹窗均支持：

- Enter 确认
- Esc 取消

---

# 13. 图标规范

整个编辑器采用统一图标风格。

建议使用线性图标（Outline Icon）。

常用图标如下：

| 功能 | 图标 |
|------|------|
| 图片 | 🖼 |
| 文字 | 📝 |
| 保存 | 💾 |
| 导出 | 📤 |
| 删除 | 🗑 |
| 锁定 | 🔒 |
| 解锁 | 🔓 |
| 显示 | 👁 |
| 隐藏 | 🚫 |
| 复制 | 📄 |
| 撤销 | ↶ |
| 重做 | ↷ |

正式版本建议统一采用 SVG Icon Library，不使用 Emoji。

---

# 14. 配色规范

整体采用浅色主题，保持简洁、现代的视觉风格。

建议配色如下：

| 用途 | 建议颜色 |
|------|----------|
| 页面背景 | #F5F5F7 |
| 画布背景 | #FFFFFF |
| 主按钮 | #3B82F6 |
| 选中状态 | #2563EB |
| 边框 | #D1D5DB |
| 文字 | #1F2937 |
| 次级文字 | #6B7280 |

避免使用高饱和颜色作为大面积背景。

---

# 15. 响应式原则

V1.0 主要适配桌面浏览器。

建议最低分辨率：

- 1440 × 900

推荐开发分辨率：

- 1920 × 1080

画布区域始终优先保证显示面积。

窗口缩小时，可压缩左右面板宽度，但不得遮挡画布。

---

# 16. UI 设计原则

整个界面设计应始终围绕"资源管理 → 画布编辑 → 对象管理 → 属性编辑"这一工作流程展开。

用户进入编辑器后，应能够在几秒钟内理解整个界面的用途，而无需学习复杂的软件概念。所有专业设计软件中的高级功能均不属于当前版本范围，界面应尽可能减少按钮数量和操作层级，让图片、文字、对象和画布成为用户唯一需要关注的内容。

本规范作为 Game Card Editor 的 UI 唯一设计标准，后续新增功能应优先保持界面一致性，而不是增加新的操作复杂度。
